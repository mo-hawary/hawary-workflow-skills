# Archive Policy

Use this when deciding whether to keep, update, merge, or archive documentation.

## Classifications

| Classification | Meaning | Default Action |
| --- | --- | --- |
| Active | Still guides current work | Keep and update only if inaccurate |
| Stale | Conflicts with source or current process | Propose edit or archive |
| Duplicate | Repeats another maintained source | Propose merge or link |
| Contradictory | Gives incompatible instructions/status | Ask or resolve with evidence |
| Historical | Useful record, no longer active | Archive with date/context |
| Unknown | Not enough evidence | Leave in place and list question |

## Archive Entry Shape

```md
## YYYY-MM-DD - <Title>

- Source:
- Reason archived:
- Replacement/current source:
- Notes:
```

## Cleanup Proposal Shape

```md
| File | Classification | Evidence | Proposed edit | Approval needed |
| --- | --- | --- | --- | --- |
```
