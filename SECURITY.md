# Security Policy

## Reporting A Vulnerability

Please report suspected vulnerabilities privately by opening a GitHub security advisory for this repository when private vulnerability reporting is available.

If GitHub does not show a private reporting option, contact the repository owner through the public profile linked from the repository.

Do not open a public issue for secrets, supply-chain concerns, or security-sensitive behavior.

## Supported Versions

Security fixes apply to the latest commit on `main`.

## Repository Safety

This repository is intended to contain public-safe Agent Skills only:

- no secrets, tokens, or credentials
- no private customer data
- no private project names
- no absolute local paths
- no implementation code from private repositories
