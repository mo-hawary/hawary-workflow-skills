# Contributing

Thanks for helping improve Hawary Workflow Skills.

## Pull Requests

All changes should go through a pull request. Direct pushes to `main` are not part of the project workflow.

Before opening a PR:

- keep canonical skill source under `skills/<skill-name>/SKILL.md`
- keep skill names lowercase and kebab-case
- keep `description` values short enough for cross-agent compatibility
- avoid private project names, customer data, absolute local paths, tokens, or secrets
- include examples under `references/` when they improve reuse without bloating `SKILL.md`

## Review

Repository ownership is defined in `.github/CODEOWNERS`. Public branch protection should require code owner approval before merging.
