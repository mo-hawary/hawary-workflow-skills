# Support

## Questions And Ideas

Open a GitHub issue for:

- skill requests
- install questions
- compatibility reports
- documentation improvements
- reproducible bugs

Use the issue templates so the request is easy to triage.

## Security

Do not open public issues for secrets, credentials, supply-chain concerns, or security-sensitive behavior. See [SECURITY.md](SECURITY.md).

## Private Project Help

This repository is public and generic. Do not share private paths, customer data, private product names, tokens, or proprietary code in issues or pull requests.
